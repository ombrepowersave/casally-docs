# 提炼维度：Demo 反馈（demo-feedback）

骨架而非死模板：维度是抓手不是死清单，原文展开到哪就挖到哪，受访者反应强烈处（惊喜、困惑、抵触）往细拆、宁细勿漏。

## 必提维度
- **逐功能反应**：对看到的每个功能 / 界面，分别记喜欢 / 困惑 / 不喜欢及原因。
- **首次理解 / 价值认知**：是否一眼看懂这产品干嘛、对谁有用；有没有 "aha" 时刻或始终没 get 到。
- **可用性问题 / 卡点**：哪里看不懂、点错、卡住、走错路。
- **功能请求 / 改进建议**：受访者主动提出想要的东西。
- **与现状对比**：相对他现在用的工具 / 做法，好在哪、差在哪、会不会换。
- **购买 / 付费意愿信号**：愿不愿付费、价格敏感度、愿付多少、什么条件下会买。

## 注意
- 聚焦受访者反应，而非访谈者的引导话术。
- 留意"嘴上说喜欢但行为 / 语气犹豫"的不一致信号。
