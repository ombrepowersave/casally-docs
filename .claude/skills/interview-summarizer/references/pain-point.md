# 提炼维度：用户痛点（pain-point）

骨架而非死模板：维度是抓手不是死清单，原文展开到哪就挖到哪，受访者反复强调、情绪强烈、讲得具体处往细拆、宁细勿漏。

## 必提维度
- **具体痛点清单**：逐条列出，每条写清现象、发生场景、触发原因。
- **痛的程度 / 紧迫性**：用原话里的情绪强度佐证（"crazy""hate""every month"）；有量化就记下来（每月多花多少钱、花多少时间精力、多频繁）。
- **触发情境 / 规律**：什么时候、什么条件下发生（季节、时段、用电事件、家庭变化等）。
- **当前应对 / 替代方案**：用了什么 app、设备、土办法；为什么不够用、卡在哪。
- **未被满足的需求 / 期望**：用户明确或隐含希望产品做到什么。
- **相关背景**：住房 / 家庭 / 设备（太阳能、EV、电池等）中与痛点直接相关的事实。

## 注意
- 访谈者是 Casally 团队成员（提问方），提炼聚焦**受访者**的表达。
- 区分"真痛点"（反复提、有具体代价）与"随口一提"。

> 深度与忠实并重，见 SKILL.md《提炼原则：又深又准》。
