# 提炼维度：硬件（hardware）

骨架而非死模板：维度是抓手不是死清单，原文展开到哪就挖到哪，受访者对设备型号、痛点、测试、集成讲得具体处往细拆、宁细勿漏。

## 必提维度
- **硬件需求 / 约束**：对接什么设备、协议、安装 / 兼容限制。
- **设备情况**：太阳能、EV、电池(Powerwall 等)、温控 / HVAC、智能电表、Span 面板等的具体型号与使用现状。
- **测试与验证 / 安全失效**：怎么测、测什么、极端场景（浪涌 / 雷击等）；失效时如何"安全地坏"（不伤下游负载）。
- **原型 → 量产**：从概念到量产的周期、工程化难点、踩过的坑。
- **安装与兼容性**：接线 / CT 钳 / 面板位 / 表前表后、是否需持牌电工与许可、上门安装的现实障碍。
- **成本与采用经济学**：硬件成本、价格、是否需补贴、谁买单、什么条件下用户才会装。
- **集成 / 协同**：设备之间如何配合工作，当前痛在哪。

## 注意
- 保留品牌 / 型号原文（如 Tesla Powerwall、Span panel）。
- 区分受访者的官方 / 已验证信息与个人观点（后者按《提炼原则》标注）。
