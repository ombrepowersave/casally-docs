## 一、产品定位：All-in-One 单相户储系统

### 核心定位
EcoFlow OCEAN 2 Plus Single Phase 的定位非常清晰——**一套单相住宅 All-in-One 户储系统**。

它把以下 7 个维度整合在同一个产品体系内解决：

| 维度 | 内容 |
|------|------|
| 光伏接入 | 3 MPPT，最高 24 kW PV 输入 |
| 储能 | LFP 电池，单机最高 60 kWh，级联最高 300 kWh |
| 全屋备电 | 0 ms 切换，覆盖全屋关键负荷 |
| EV 充电 | 家庭电动车充电集成 |
| 动态电价优化 | HEMS 自动充放策略 |
| 智能家居联动 | EcoFlow App + Power Insight 平板 |
| 安装交付效率 | 预集成设计、内置 smart meter 和 gateway |

### 作者观点
> "EcoFlow 想把竞争重点从单个设备参数，推进到家庭能源系统体验。"  
> 作者认为**这个方向是对的**。OCEAN 2 Plus 更像一个**家庭能源总控系统**——硬件是入口，软件、数据预测、负荷调度、备电切换和生态联动才是后面的价值。

**战略意义：** 这标志着 EcoFlow 从"卖设备"向"卖系统体验"的转型。产品的竞争维度从单点参数（如电池容量、逆变器功率）升级为系统性体验（如安装便利性、智能调度、生态联动）。

## 二、发电侧：3 个独立 MPPT，解决复杂屋顶痛点

### 关键参数

| 参数 | 数值 |
|------|------|
| 每串最高 PV 输入 | 8 kW |
| 总 PV 输入 | 最高 24 kW |
| MPP Tracker | 3 个独立 |
| 每路 MPPT 配置 | 1 路 String |

### 产品逻辑分析
**3 个独立 MPPT 是这套产品的重要卖点。** 作者指出，这一设计非常贴近欧洲和澳洲住宅屋顶的现实情况：

**海外独栋住宅屋顶的典型特征：**
- **多朝向：** 屋顶有东、南、西等多个坡面，不同朝向的光伏组件发电曲线完全不同
- **局部遮挡：** 烟囱、天窗、树木等造成局部遮挡，影响整串组件
- **屋顶面积碎片化：** 不规则屋顶分割成多个可安装面
- **安装面不规则：** 三角形、梯形等形状限制组件布置

**3 个独立 MPPT 的实际价值：**
- 不同朝向的光伏组件分别独立优化，互不影响
- 局部遮挡时，只有被遮挡的那一路受影响，其他路正常工作
- 最大化屋顶可用资源的利用率

> 作者点评：这比单纯讲"24 kW PV 输入"更容易打动实际用户。

## 三、储能侧：大容量扩展，瞄准高能耗家庭

### 关键参数

| 参数 | 数值 |
|------|------|
| 单台逆变器最大电池模块数 | 12 个 |
| 单机最高储能容量 | **60 kWh** |
| 并网场景最大级联数 | 5 台 |
| 系统总容量上限 | **300 kWh** |
| 单电池模块标称容量 | OCEAN 2 LFP Battery 5 kWh（标称 5.02 kWh） |

### 瞄准的用户画像
作者分析认为，**60 kWh 单机扩展上限**瞄准的是**复合型用户**：

> **典型画像：大屋顶光伏 + 高家庭负荷 + 备电焦虑 + 动态电价套利需求**

**澳洲市场的特殊性：**
- 独栋住宅屋顶面积大，光伏装机潜力高
- 空调用电负荷高
- EV（电动车）渗透率正在快速提升
- 家庭用电结构已在变化：未来会有更多家庭同时拥有 **EV、热泵、泳池泵、电炉、烘干机、空调**等高耗能设备

### 关键策略：分阶段投资
> **用户可以先装一个基础容量，等后面买 EV、加热泵、家庭用电增加，再逐步扩容。**

这意味着户储的选型逻辑正在发生变化：

| 传统选型逻辑 | 新选型逻辑 |
|--------------|------------|
| 家庭一天用多少电 | 光伏装机规模 |
| 简单匹配容量 | 电价结构（峰谷价差/动态电价） |
| 一次性到位 | EV 使用频次 |
| — | 备电时长要求 |
| — | 可调度负荷比例 |

## 四、备电侧：全屋备电是高端户储的分层点

### 核心卖点
OCEAN 2 Plus 重点强调 **Integrated Whole-Home Backup**（集成全屋备电）。

### 覆盖的备电设备清单

| 类别 | 设备 |
|------|------|
| 出行 | EV Charger |
| 供暖/热水 | Heat Pump、Immersion Heater |
| 清洁/泳池 | Swimming Pool Pump |
| 厨房 | Microwave、Electric Stove、Dishwasher、Refrigerator、Freezer |
| 洗衣 | Dryer、Washing Machine |
| 照明 | Lights |
| 通信/办公 | Computer、Wi-Fi Router |

这些负荷覆盖了**厨房、供暖、通信、冷藏、洗衣、出行**六大生活场景。

### 0 ms 切换的真实边界
> 作者特别澄清："0 ms 切换"需要满足条件，包括：
> - **当地电网法规**允许
> - **电网开路状态**符合要求
> - **总负载低于额定备电输出功率**
> - **电网条件稳定**
>
> 更稳妥的说法是：**在规定条件下实现无感切换。**

### 安装商视角：真正的技术难点
从安装商角度看，全屋备电真正难的地方在于：

| 技术难点 | 说明 |
|----------|------|
| 切换逻辑 | 并网/离网切换时序、相位同步 |
| 浪涌承受能力 | 电机类设备（空调/热泵/水泵）启动电流 |
| 负荷优先级管理 | 有限容量下如何分配关键负荷 |
| 孤岛保护 | 防逆流、防孤岛效应 |
| 配电箱改造 | 备电回路与非备电回路分离 |
| 并网合规 | 当地电网公司的技术审查要求 |

### EcoFlow 的做法
EcoFlow 将 **smart meter（智能电表）和 gateway（通信网关）内置**到系统中，价值在于：
- 减少外部箱体
- 降低接线复杂度
- 减少安装出错概率

## 五、智能侧：HEMS + 动态电价，让电池具备资产运营属性

### Intelligent SmartEarning Mode
EcoFlow 的 HEMS（家庭能源管理系统）核心功能是 **SmartEarning Mode**——根据动态电价自动决策：

| 场景 | 系统行为 |
|------|----------|
| 电价低 | 从电网充电（储能） |
| 电价高 | 优先用电池放电（自用或馈网） |
| 白天光伏多发 | 存入电池 |
| 系统自动决策 | 判断何时充、何时放、何时省 |

### 核心洞察
> **户储的价值，正在从备用电源延伸到家庭能源资产运营。**

在动态电价市场里，用户买电池的理由变得更丰富：
1. 白天光伏多发 → 存起来
2. 电价低的时候 → 充进去
3. 电价高的时候 → 优先用电池
4. 系统**自动判断**什么时候该充、什么时候该放、什么时候该省

### 作者比喻
> "这个逻辑有点像家庭能源里的'自动驾驶'。用户每天盯电价、看天气、算负荷很麻烦，系统通过 HEMS 自动做决策，才是智能化真正有价值的地方。"

这意味着 EcoFlow 的智能策略正在将电池从"被动备用设备"转变为"主动运营资产"——在电价波动中为用户创造经济收益。

### 战略价值分析
作者对这一点的分析非常深入。他指出：

> "Power Insight 看起来是一个配件，本质上是 EcoFlow 在争夺 **家庭能源交互入口**。"

**为什么需要一块墙面屏幕？**
- 户储产品安装后，用户打开 App 的频率会越来越低（这是行业通病）
- 但如果家里墙上有一块屏幕，能源数据**长期可见**
- 家庭成员都能看到：今天发了多少电、用了多少电、存了多少电、卖了多少电
- 用户对这套**系统的感知会明显变强**

**消费电子行业的经验：**
> 能被用户高频看见的屏幕，往往会成为生态入口。

**Power Insight 的长期价值可能体现在：**
| 价值维度 | 说明 |
|----------|------|
| 用户注意力 | 持续占据家庭视觉焦点 |
| 能源数据 | 积累家庭用电行为数据 |
| 设备控制 | 可扩展为全屋能源控制中心 |
| VPP 服务沉淀 | 为未来虚拟电厂（VPP）提供交互入口 |

## 十、容量扩展方式：30 kWh 以上新增电池堆

### 扩展架构
- 现场展示标准配置为 **30 kWh stack**
- 当容量需要超过 30 kWh 时，可在旁边**新增一组电池堆**

### 扩展技术细节

| 扩展步骤 | 说明 |
|----------|------|
| 连接方式 | 新增电池采用 **DC coupled**（直流耦合） |
| 接线方式 | 新增电池堆顶部配置接线盒 |
| 接入方式 | 从侧边接入原有逆变器 |
| 统一管理 | 单台逆变器统一管理最高 **60 kWh** 储能容量 |

### 技术点评
> "DC coupled 通常有利于光伏到电池的能量转换效率和系统统一管理，但实际效率还是要看逆变器拓扑、BMS 策略和具体运行工况。"

### 真正的挑战：BMS 能力
> "容量做到 60 kWh 后，真正考验的是系统管理能力。"

用户看到的是"再加一组电池"，但系统背后要解决的复杂问题包括：

| 挑战 | 说明 |
|------|------|
| **多模块老化不均** | 新旧电池堆内阻、容量衰减不同 |
| **安全平衡** | 不同老化状态的电池在充放电时的热管理 |
| **效率平衡** | 如何分配充放电功率以使整体效率最优 |
| **寿命平衡** | 避免老龄模块被过度充放而加速淘汰 |
