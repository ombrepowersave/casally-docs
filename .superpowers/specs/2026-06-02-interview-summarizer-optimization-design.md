# 设计文档：`interview-summarizer` 技能优化

日期：2026-06-02
状态：已确认，待实现
背景：技能经多次增量更新（新增"忠实原文""转录纠错"两段约束 + 内联提醒），文本出现拼接、重复、否定式语气。用户担心约束压制总结深度，并希望做一次整体梳理 + references 补厚。

## 目标
1. 重写 SKILL.md：消除增量拼接痕迹，把约束改成正向、连贯的表达，**确保防幻觉的同时不压制提炼深度**。
2. 补厚 references：保持"骨架非模板"理念，给偏薄主题补维度抓手。
3. 用 A/B 实测验证优化后深度不倒退、无幻觉。
4. **规则行为基本不变**，只优化表达；唯一行为增强是重命名步骤兼容 tracked/untracked。

## 第 1 块：SKILL.md 结构与约束重写

现状问题：
- 两段约束夹在简介与流程之间，读者先读"别做什么"才到"怎么做"。
- 步骤 2b 重复复述忠实原文约束。
- 约束整体是否定式语气，是"担心压制深度"的来源。

改法：
- 合并两段约束为一节 **《提炼原则：又深又准》**，正向框架：
  - **深**：往细拆、宁细勿漏——把原文已有细节挖透。
  - **准**：忠实原文（不臆造/不补全/不外推；区分"说过"与"推断"并标注；金句一字不差；宁缺勿补；可追溯）+ 转录纠错（只改明显错、不动 `.txt`、关键纠错标注、不借机新增事实）。
  - 关键点睛句："约束的是*别编没说过的*，不是*少写*；原文已有的细节要尽量挖透。" 直接回应"约束压制深度"的顾虑。
- 步骤 2b 删除重复复述，改为一行指针："按维度提炼，遵循《提炼原则》"。
- 统一术语、顺措辞，使全文连贯。**规则不变**（只改明显转录错、不动原始 `.txt`、关键纠错标注、不新增事实）。

## 第 2 块：references 补厚（保持"骨架非模板"）

- 5 个文件统一格式：标题 → 一句"骨架非模板"提示 → 维度清单 → 注意事项 → 一行指向 SKILL.md《提炼原则》（不重复全文约束，避免 5 处重复）。
- 补薄维度（示例）：
  - `hardware.md`（现仅 2 维）：加「测试与验证方法 / 安全失效行为」「原型→量产周期」「安装与兼容性」「成本与采用经济学」。
  - `energy-expert.md`：加「合规 / 认证路径」「监管 / 政策门槛与成本」。
  - `pain-point.md` / `demo-feedback.md` / `generic.md`：查漏补缺、措辞对齐，不大动。
- 维度仍是"骨架"，给抓手不给死清单。

## 第 3 块：重命名兼容 tracked / untracked

语义澄清：仓库中部分访谈已 git 入库（tracked），部分是新加入文件夹的（untracked）。步骤 2e 重命名要自动判断：
- 文件已被 git 跟踪（`git ls-files` 命中）→ 用 `git mv`（保留历史）。
- 未跟踪 → 用普通 `mv`。
- `.txt` 与 `.md` 仍共用同一规整基名。

## 第 4 块：A/B 实测验证

- 对照组：`interviews/s5-Hardware1.0/2026-05-24-Harry Jackson-hardware.md`（加约束前跑的、用户已认可、内容最丰富）。
- 实验组：用优化后的技能在同一原稿 `2026-05-24-Harry Jackson-hardware.txt` 上重跑，产出写到 `/tmp`（不覆盖仓库旧版）。
- 对比判据：深度保持或更好、无幻觉、转录纠错合理、结构更顺。
- 不达标（深度倒退）→ 回调《提炼原则》措辞再跑。
