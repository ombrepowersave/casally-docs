# 设计文档：`interview-summarizer` 技能

日期：2026-06-02
状态：已确认，待实现

## 1. 用途

扫描本项目 `interviews/` 目录下的访谈逐字稿（`.txt`），按访谈主题用相应维度提炼成结构化的中文摘要（`.md`），并把杂乱的原始文件名规整为统一格式。

下游知识库工具 gbrain 通过文件格式区分内容：**只读 `.md` 摘要，忽略 `.txt` 原始逐字稿**。

## 2. 背景与素材特征

- `interviews/` 下有 5 个分组目录，约 70 份访谈，主题各不相同。
- 每个文件是一场访谈逐字稿：带录音时间戳、双人对话（访谈者为 Casally 团队成员，如 Amber Fu + 受访者），围绕 Casally 这款帮用户省电费的产品。
- 逐字稿很长（数百行）、口语化、含大量寒暄与噪音。
- 文件名缺乏统一命名规划。

## 3. 技能结构（方案 2：流程 + 每主题独立参考文件）

```
interview-summarizer/
├── SKILL.md                  # 主流程 + 规则
├── scripts/
│   └── scan.py               # 扫描：列出"无同名 .md 的 .txt"、解析文件名日期
└── references/
    ├── pain-point.md         # 痛点访谈 细化维度
    ├── demo-feedback.md      # Demo 反馈 细化维度
    ├── energy-expert.md      # 专业咨询 细化维度
    ├── hardware.md           # 硬件 细化维度
    └── generic.md            # 通用兜底（未知主题）
```

**可更新性**：新增主题 = 加一个 `references/xxx.md` + 在 SKILL.md 主题映射表加一行，不动主流程。`generic.md` 作为未知主题的天然兜底。

## 4. 主流程（SKILL.md）

1. **确定范围**：支持三种粒度
   - 单文件：只总结指定的那一份
   - 单目录：处理某个 `sX-xxx/` 下所有访谈
   - 全部：扫描 `interviews/` 下所有目录
2. **跳过 or 重做**：
   - 目录级 / 全部级：若已存在同名 `.md` 则**跳过**（批量增量更新）
   - 单文件级：**强制重做**（覆盖重新生成）
3. **识别主题**：先看所在 `sX-xxx/` 目录前缀映射 → 再读正文内容确认 / 校正 → 都不匹配则用 `generic.md`
4. **深度提炼**：以对应 reference 的维度为骨架，但顺着受访者**重点强调处往细里拆，宁细勿漏**；不死套模板。
5. **写摘要 `.md`**：中文为主，英文金句保留原句。
6. **规整命名**：把 `.txt` 与新生成的 `.md` 都改成 `YYYY-MM-DD-访谈人-主题`
   - 日期：解析自原文件名（如 `...20260524...` → `2026-05-24`）；正文录音时间戳不作日期；解析不到则省略日期段。
   - 访谈人：保留原名（多为英文），去掉文件名中的额外备注（如"（Span员工）""序号-"前缀），相关身份信息在摘要正文中体现。
   - 主题：英文 slug。
   - `.txt` 与 `.md` 共用同一规整基名，配对一致。

## 6. 各主题提炼维度

### 通用维度（所有主题）
- 受访者画像：姓名、背景；如有住房 / 家庭 / 地区 / 设备（太阳能、EV、电池、温控等）
- 主题判定 + 一句话依据
- 关键金句原话（保留英文原句，标注说话人）
- 行动项 / 机会点

### 用户痛点（pain-point）
- 具体痛点清单（现象、场景、触发原因）
- 痛的程度 / 紧迫性（用户原话里的情绪强度）
- 当前应对 / 替代方案（用了什么 app、土办法）
- 未被满足的需求 / 期望

### Demo 反馈（demo-feedback）
- 对每个功能的具体反应（喜欢 / 困惑 / 不喜欢）
- 可用性问题、卡点
- 功能请求 / 改进建议
- 购买 / 付费意愿信号

### 专业咨询（energy-expert）
- 专业观点 / 行业知识点
- 对产品方向的专业建议
- 提到的技术 / 政策 / 术语（电价、净计量、储能等）

### 硬件（hardware）
- 硬件相关需求 / 约束
- 设备情况（太阳能、EV、电池、温控等）

### 通用兜底（generic）
- 仅用通用维度，并尽量从内容中识别该访谈实际关心的主线，按内容自适应展开。

## 7. 摘要 `.md` 模板（骨架，可按内容深挖）

```markdown
# [访谈人] · [主题] · [日期]

## 受访者画像
（背景、住房/家庭/地区；如有：太阳能/EV/电池/温控等设备情况）

## 主题判定
判定主题 + 一句话依据

## 核心提炼
（按该主题 reference 的维度展开，顺着受访者重点往细拆）

## 关键金句
- "原英文 quote" —— [说话人]

## 行动项 / 机会点
- …
```
