### 逐功能反应

**旧版主屏（Home）**
- Natalie 一眼读出的元素："usage and saving"（用量与省钱）、底部栏的 "apply now"、以及 "power outage warning"（停电警告）。
- 喜欢看到 saving 数字：当看到 $268.81 时主动说这是"本月的省钱"，并强调"这正是我日常也想看到的东西，因为我真的很想省很多钱"。
- 对主屏整体满意："this works better because I can see the usage of my daily basis…how much bill I've used for today and how much I've saved for today…the home page is okay with me."
- 对停电警告：她假设会有推送通知（"I'm pretty sure there's a notification that will open my phone"）提醒她——这是她的**假设/期待**，原文未由产品确认具体推送机制（推断她默认存在）。
- 问"主屏没有缺什么想要的吗"，她答没有（"the home page is okay with me"）。

**事件页 / EV charging during high price（高价时段充电事件）**
- 能读懂事件结构：发生了什么（EV 在电价高于平时充电）、为什么重要、影响（在三小时内改排充电可减少开支）。
- 正面："that will work so good for me because I can monitor…I can see what happened…the impact when I charge my car."
- 对事件卡底部的 mute / remove 按钮提问其作用；得到解释（静音=不再通知、删除=移除事件）后表示接受："it's okay with me…this works perfect for me and I don't have any issue with it."

**储蓄页（Saving，旧版）**
- 自行把数字串起来理解：举例"以前一个月可能用 $412"，主屏显示省 268，于是"估计省了 268…我付账单会比平时低…对我太完美了"。注意：$412 是她**自己举的假设性例子**，非系统给出的数字。
- 展开 EV 高价事件的 bottom sheet（底部信息条）时，她一开始把"high-price 时段"理解为"早上不用太阳能、改用电网、所以价格变高的时段"，并主动问"Is that what this means?"——说明该底部条文案她**没有第一时间读懂**，需要追问。
- 访谈者澄清 bottom sheet 的本意是"告诉你当前正在用太阳能、今天到目前已省约 $5"后，她说"Now I get it"——属于经解释后的理解，而非首次自解。

**Profile / 数据与连接（utility、设备连接）**
- 访谈者请求连接她的电力公司（utility）账户以核算上月花费与节省。她没有正面回答"是否授权连接 utility 账户"，而是把话题转向**设备在线状态**："一旦我给 EV 充电，它就会显示在 app 上、在线且已连接"，并反复确认 app 会显示 EV / 设备 online 或 offline——得到肯定后说"Ah that's perfect"。
- 对连接 solar / power wall / EV 到 app 表示 OK（"Yeah…that's perfect / sure"）。

**添加家庭成员（Add family members）**
- 主动提问："能不能把邀请分享给家人以外的人？"（"can I share this invitation to someone else apart from my family?"）——显示她关心分享范围。
- 访谈者答：对方也下载 app，在"一个账户、一个家庭"下使用。她回"All right…it's okay with me"。

**改版主屏（新版 Home）**
访谈者讲解了一版较大改动，心智从"优化用电"转为"守住你能接受的预算边界"。新版包含：
1. 顶部一行 **overpay risk today（今日超支风险）**，例 $86，含义"今天若什么都不做，可能超预算"（非最终账单）。
2. **possible factors（可能原因）**，用大白话给出最主要原因（如 EV charging）。
3. **monthly saving target（每月省钱/预算目标）**，例 $100；强调不是越低越好，而是"你能舒适接受的账单水平"；目标由历史账单 + 家庭配置（house size、是否有 EV/solar/battery）推算，不让用户随便填数字。
4. **plan card（计划卡）**——访谈者称这是最重要的改动：saving 告诉你风险，plan 告诉你怎么做（如"现在改 EV 充电时段可省约 $12，帮你守住 $100/月目标"，并给时间窗）。执行有两种安全方式：打开 Tesla app 自行确认，或经授权自动执行。
5. **outage warning + backup runtime（停电警告 + 备用续航时长）**，例如约 6 小时，帮你判断"现在洗衣 / 推迟 / 能否安心继续工作通话"。
6. **device status（设备状态）**：绿=在线、红=离线，离线立即提醒（缘起她曾"半天后才发现系统停了"）。
7. **usage** 降为次要信息，仍保留但不作为主要提示。

Natalie 对新版的总体态度："I prefer the new one because it has a lot of features that I I did expect to have…It's perfect."
- 她认为新版**最有用的部分是 usage 和 saving**（"the usage and the saving part"）。
- 对 **goal/预算目标**：明确正面，并主动给出自己想设的目标——"想把电费目标设成比如 $80"（"set a target like let's say $80"），理由是不想超预算（"I don't want to go beyond my budget"）。
- 对 **plan（每日省钱建议）**：明确正面，"that will be so helpful for me"。
- 注意：演示新版过程中，她多次插入与内容无关的转录噪声词（"Heat. Heat."、"Rex"、"What is this?"），且对 backup runtime 那段访谈者讲解时转录显示她说了 **"Don't care."**（line 537）。这句若属实，可能是对"停电备用续航"功能**不太在意**的信号；但考虑她此前明确说过担心停电影响工作通话，这与"Don't care"存在矛盾，故标注为**可能的不一致/转录存疑信号**，不作定论。

**改版储蓄页（新版 Saving）**
访谈者讲解：Saving 页重定位为"风险暴露页"，展示是什么把你推离目标、你能避免什么。
- 顶部数字例 **$312**，含义"通过调整这些行为最多可省约 $312"。
- 下方柱状图说明数字不是随机的、与近几个月检测一致；不喜欢可关闭（"totally an option"）。Natalie 说"I don't mind, it's okay"。
- 按 **类别拆解**（EV / solar / power wall），看出哪几项最"费钱"——给出三大驱动项：高价时段 EV 充电、power wall 充电、太阳能可用性。Natalie 正面回应："that works perfectly because…if I have three in my home, so I can monitor them and I can know what particular time to charge them and I can see how much I'm saving."

- **checklist（数据接入清单）**：列出已接入信息（如 solar export 数据、utility 账单等），未接智能电表则缺失对应部分。Natalie 回"Okay. Okay."（中性）。

## 行动项 / 机会点
- **强化 usage + saving 仍是核心钩子**：这是 Natalie 唯一持续高热度的部分，新旧版她都点名"most useful"。改版应确保省钱数字与每日/每月追踪保持显眼。
- **打磨 bottom sheet 文案**："正在用太阳能 / 今日已省 $X" 的实时条被误解为"高价时段说明"，需让"当前是否在用免费太阳能、已省多少"一眼可读。
- **可自定义预算目标是真实需求**：用户想自己设目标（如 $80）并被系统帮助守住；确认目标既能系统推算、也允许用户调整。
- **设备 online/offline + 充电中状态**：明确产品确实会实时显示并在离线时即时提醒，正中其痛点（她曾半天后才发现系统停摆）。
- **澄清停电/备用续航价值**：她对 backup runtime 的当场反应偏冷（"Don't care"存疑），但背景里她担心停电影响工作通话——需在真实试用中验证该功能对她到底重不重要。
- **明确 utility 账户授权动作**：她对"连接 utility 账户"始终未正面答应（只谈设备在线显示）。后续需单独确认她是否愿意授权 utility 账号、以及对智能电表接入（影响 checklist 完整性）的接受度。
- **家庭/分享范围**：她想邀请家人以外的人；"一账户一家庭"的限制可能与其期望存在落差，值得跟进。
- **付费转化**：付费意愿正面但以"确实省钱"为前提，且未给价位；建议在真实试用后用实际节省数据再谈定价与愿付金额，并留意其"全程附和"可能高估热情。
- **信任/审计类模块（trust layer、bill tab、bill proof、checklist）**：她反馈普遍中性，未感知到与省钱同级的价值；可考虑这些模块的呈现优先级与触发时机（默认收起、按需展开）。
